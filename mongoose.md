MongoDB is a NoSQL, document-oriented database designed for scalability, high performance, and ease of development. Unlike traditional relational databases, MongoDB stores data in flexible, JSON-like documents.

-------------------------------------------------------------------------------------------------

 Key Features:
 ---------------
Schema-less: Documents can have different fields.

Uses BSON (Binary JSON) for storage.

Supports nested documents and arrays.

High availability via replication.

Horizontal scalability via sharding.

-------------------------------------------------------------------------------------------------

What is an ORM?  
----------------- 

An ORM (Object-Relational Mapping) is a tool that allows developers to interact with a database using an object-oriented paradigm. It abstracts away SQL queries and lets you manipulate data using your programming language's native syntax.

-------------------------------------------------------------------------------------------------

What is Mongoose?
------------------

Mongoose is an ODM (Object Data Modeling) library for MongoDB and Node.js. It provides a schema-based solution to model application data and includes built-in type casting, validation, query building, and business logic hooks.

-------------------------------------------------------------------------------------------------

Key Features of Mongoose:
---------------------------
Schema definitions for collections.

Middleware support (e.g., pre/post hooks).

Built-in data validation.

Easy relationship modeling via populate().

Connection and session handling.

-------------------------------------------------------------------------------------------------

When to Use Mongoose?
------------------------

Use Mongoose if:
------------------
You want to enforce a schema on MongoDB documents.

You prefer validation and middleware support.

You need to use powerful query-building and population features.
