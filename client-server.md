Client-Server Architecture : 

Client-server architecture is a foundational model in network computing where the workload is divided between service providers (servers) and service requesters (clients). This architecture facilitates communication and resource sharing over a network.

Components :

Client:
The client is the requester of services or resources. It initiates communication by sending requests to the server. Clients are usually user-facing applications like web browsers, mobile apps, or desktop software.

Server:
The server is a centralized system that listens for client requests, processes them, and returns appropriate responses. Servers provide resources, data, or services such as hosting websites, managing databases, or running applications.

Working : 

Request: The client sends a request to the server over a network (often the Internet or a local network).

Processing: The server processes the request, which may involve querying databases, performing computations, or accessing files.

Response: The server sends the requested data or status back to the client.

Rendering: The client receives the response and presents the information or service to the user.
